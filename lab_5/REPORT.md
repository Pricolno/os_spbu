## Exersice 6
>Запустите программу со случайным интервалом прерываний: ./x86.py -p looping-race-nolock.s -t 2 -M 2000 -i 4 -r -s 0, задавая разные начальные значения (-s 1, -s 2 и т. д.). Можете ли вы, глядя на чередование потоков, сказать, каким будет конечное значение value? Влияет ли на чтонибудь интервал между прерываниями? Где возникновение прерывания безопасно? А где нет? Конечно, можно понять значение по адресу 2000, но оно будт завист от прерываний.  В критической секции может просходит перенос значения в регистр и инкремент, а потом может подключится другой поток и тоже выполнить 3 инструкции и тогда ожидаемое прибавление 2 не будет.
```shell
# critical section
mov 2000, %ax  # get 'value' at address 2000
add $1, %ax    # increment it
mov %ax, 2000  # store it back
```

## Exersice 7
>Теперь рассмотрим фиксированные интервалы между прерываниями:
./x86.py -p looping-race-nolock.s -a bx=1 -t 2 -M 2000 -i 1. Чему равно конечное значение разделяемой переменной value? А что, если задать флаги -i 2, -i 3 и т. д.? При каких интервалах между прерываниями программа дает «правильный» ответ?

Значения равны соответственно 1, 1, 2. Для %bx = 1 достаточно хотябы 3, но для произвольного %bx, нужно чтобы %bx был кратен 3 потому что в повторяющемся цикле 6 элементарных операций. 

```shell
# critical section
mov 2000, %ax  # get 'value' at address 2000
add $1, %ax    # increment it
mov %ax, 2000  # store it back

# see if we're still looping
sub  $1, %bx
test $0, %bx
jgt .top	
```

## Exersice 9
>И последняя программа: wait-for-me.s. Запустите ее следующим образом:
./x86.py -p wait-for-me.s -a ax=1,ax=0 -R ax -M 2000. При этом программа
записывает в регистр %ax значение 1 в потоке 0 и значение 0 в потоке 1,
а затем наблюдает за регистром %ax и ячейкой памяти по адресу 2000.
Как должен вести себя код? Как значение по адресу 2000 используется
потоками? Каким будет конечное значение?

Если %ax = 1, то переход в .signaller и запись 1 в 2000. Иначе, переход в .waiter и бесконечный цикл пока в 2000 не будет 1. Но первый поток записал 1 в 2000 и второй поток завершается.

```shell
./x86.py -p wait-for-me.s -a ax=1,ax=0 -R ax -M 2000

2000      ax          Thread 0                Thread 1         
    0       1   
    0       1   1000 test $1, %ax
    0       1   1001 je .signaller
    1       1   1006 mov  $1, 2000
    1       1   1007 halt
    1       0   ----- Halt;Switch -----  ----- Halt;Switch -----  
    1       0                            1000 test $1, %ax
    1       0                            1001 je .signaller
    1       0                            1002 mov  2000, %cx
    1       0                            1003 test $1, %cx
    1       0                            1004 jne .waiter
    1       0                            1005 halt


```
